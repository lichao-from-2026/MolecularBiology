# RsPress 构建规则

## 1. 核心命令

- 开发命令：`npm dev`
- 构建命令：`npm build`
- 预览命令：`npm preview`

## 2. 目录结构规则

### 2.1 文档存放

- 文档存放于 `docs/` 目录
- 按分类建立子文件夹

### 2.2 命名规则

- 同层级：递增序号 + 中文（如 `01-中文标题`）
- 不同层级：序号重置为 `01-`

### 2.3 结构示例

```
docs/
├── 01-基础理论/
│   ├── 01-Agent概念/          # 对应知识点有多篇文档时，需要创建文件夹囊括。
│   │   │── 01-Agent定义与概念.md
│   │   └── 02-Agent XXXX.md
│   ├── 02-核心组件/
│   │   │── 01-Agent核心组件.md
│   │   └── 02-Agent XXXX.md
│   └── 03-XXXX.md            # 对应知识点仅有一篇文档时，仅需创建md文档，无需创建文件夹。
└── 02-技术栈详解/
...
```

## 3. 导航管理规则

### 3.1 使用 \_nav.json 管理导航

- 必须使用 `_nav.json` 管理侧边栏导航
- 在 `docs/` 根目录放置主导航文件
- `rspress.config.ts` 中不应该硬编码 sidebar 配置

### 3.2 nav.json 结构规范

```json
[
  {
    "text": "导航名称",
    "link": "/路径",
    "items": [
      {
        "text": "子项",
        "link": "/子路径"
      },
      {
        "text": "分组",
        "items": [...]  
      }
    ]
  }
]
```

### 3.3 导航层级与格式规范

#### 3.3.1 导航层级格式

| 导航层级 | 格式示例 |
|---------|--------|
| 一级导航 | `"text": "一、基础知识"` |
| 二级导航 | `"text": "1. 分子生物学入门"` |
| 三级导航 | `"text": "1. DNA结构与功能"` |
| 延伸导航 | `"text": "1.1 具体子项"` |

#### 3.3.2 编号规则
- 一级导航：中文数字编号（一、二、三...）
- 二级导航：阿拉伯数字编号（1.、2.、3...）
- 三级及以上导航：阿拉伯数字嵌套编号（1.1、1.2、2.1...）
- 同层级编号连续不重复
- 不同层级编号重置为 1 | 1.1

#### 3.3.3 路径规范
- 目录路径以 `/` 结尾
- 文件路径直接指向文件，不加 `/`
- 路径使用相对路径，不含扩展名

#### 3.3.4 链接设置
- 一级导航链接指向分类默认文件
- 二级导航在无三级导航时指向对应文件
- 三级导航链接必须指向具体文件

#### 3.3.5 激活匹配
- 一级导航设置 `activeMatch` 确保分类下所有页面激活对应导航

### 3.4 \_nav.json 字段说明

| 字段        | 类型      | 必需 | 说明          |
| --------- | ------- | -- | ----------- |
| text      | string  | 是  | 导航项显示文本     |
| link      | string  | 否  | 导航项链接地址     |
| items     | array   | 否  | 子导航项数组      |
| collapsed | boolean | 否  | 是否默认折叠（仅分组） |

## 4. 重要注意事项

- 文档使用 Markdown 格式（.md），仅在需要自定义组件时使用 MDX
- 导航配置使用 `_nav.json` 管理，不硬编码在 `rspress.config.ts`
- 输出 100% 纯净的 Markdown 文本格式，不得包含任何可能干扰解析的非标准内容
- 保持 docs/ 目录层级结构清晰有序
