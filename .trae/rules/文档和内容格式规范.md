# 文档和内容格式规范

## 1. 文档格式铁律

### 1.1 强制标题格式

必须区分文档内容格式（本文规则）和 docs 下文件夹格式（RsPress 构建规则）。

- 一级标题：`# 01-XXXX`
- 二级标题：`## 1. XXXX`
- 三级标题：`### 1.1 XXXX`
- 四级标题：`#### 1.1.1 XXXX`

### 1.2 格式规则

- 标题层级严格对应 Markdown 层级
- 编号连续、不跳级、不重复、不乱序
- 段落之间空一行，重点内容加粗
- 英文术语首次出现必须标注中英文全称
- 步骤用有序列表，知识点用无序列表，对比用表格
- 使用 RsPress 内置组件：`:::tip` / `:::warning`
- 所有文档必须使用 包含 **Frontmatter 的 Markdown**  格式（.md）。仅在需要自定义组件时使用 MDX
- 禁止：复杂内嵌 HTML、非标准 Markdown 语法、混乱嵌套

## 2. 知识内容规范

### 2.1 内容规范

- 术语统一：中英文全称+缩写（首次出现必须标注，如 LLM (Large Language Model)）
- 易混淆概念必须用表格对比
- 严禁：编造数据、虚构实践、非权威结论

### 2.2 文档结构示例
```
---
title: 文档标题
description: 文档简短描述
author: 作者名
date: YYYY-MM-DD
category: 分类
---

# 01-标题（按序递增）

## 1. 二级标题（按序递增）

```
## 3. 图片下载处理规则

### 3.1 下载能力判断

| 能力 | 说明 |
|-----|------|
| ✅ 能下载 | 直接可访问的URL（无需登录、无付费墙） |
| ❌ 不能下载 | URL无效、需要登录权限、网站限制反爬 |

### 3.2 处理流程

**情况1：能成功下载**
- 直接下载到本地 `public/images/` 对应分类文件夹
- 自动更新文档中的图片引用路径

**情况2：下载失败**
- 在文档对应位置标注 `:::warning 找不到原始图像`
- 注明原始引用地址和出处信息
- 输出失败列表供用户手动处理

**情况3：用户提供新图片**
- 接收用户提供的有效图片URL或本地路径
- 保存到 `public/images/` 对应分类
- 更新文档中的引用路径

### 3.3 用户配合要求

- 学术图片建议从 PMC、BioRxiv、Jove 等开放获取平台获取
- 优先使用 Creative Commons 授权图片
- 手动下载后提供图片路径或URL
